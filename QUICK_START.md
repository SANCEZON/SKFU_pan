# Быстрый старт деплоя

## Ответ на ваш вопрос

**Нужен ли VPS?** 

**Нет, не обязателен**, но **рекомендуется** для удобства.

### На обычном хостинге (shared hosting) можно:
✅ Разместить фронтенд
✅ Использовать MySQL базу данных
✅ Использовать phpMyAdmin

❌ **НЕЛЬЗЯ** запустить Node.js API (нет поддержки)

### Решения:

1. **Взять VPS** (~200-300₽/мес) - все работает как сейчас
2. **Использовать внешний API** (Railway/Render бесплатно) - API на другом сервере

## Что уже готово

✅ Файл `.htaccess` для SPA
✅ Инструкции в `DEPLOY_STEPS.md` и `DEPLOY_SHARED_HOSTING.md`
✅ Настройка CORS в API для внешних запросов

## Что делать дальше

### Вариант 1: Взять VPS (проще всего)

1. В панели TimeWeb → "VDS/VPS Серверы"
2. Создайте VPS (минимальная конфигурация)
3. Следуйте плану деплоя для VPS (все готово)

### Вариант 2: Использовать внешний API (бесплатно)

**Шаг 1:** Разместите API на Railway.app (5 минут)
- Зайдите на https://railway.app
- Подключите GitHub репозиторий
- Укажите папку `api/`
- Настройте переменные окружения (см. `DEPLOY_STEPS.md`)

**Шаг 2:** Настройте БД на TimeWeb (5 минут)
- Создайте MySQL базу в панели
- Импортируйте `database/schema.sql` через phpMyAdmin

**Шаг 3:** Соберите и загрузите фронтенд (5 минут)
```bash
# Локально
echo "VITE_API_URL=https://your-api.up.railway.app" > .env.production
pnpm build --mode production

# Загрузите dist/ на TimeWeb через FTP
```

**Подробная инструкция:** см. `DEPLOY_STEPS.md`

## Рекомендация

Если бюджет позволяет - **берите VPS**. Это проще и надежнее.

Если нет - используйте **Railway.app** для API (бесплатно, работает отлично).
